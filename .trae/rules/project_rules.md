# trade-alpha 项目规则

## 项目概述
- **项目名称**：trade-alpha
- **项目类型**：股票数据分析与交易系统
- **Python 版本**：3.14+
- **许可证**：Apache License 2.0

## 核心原则

### 1. 设计文档优先
- 先设计后实现
- 根据任务复杂度自行判断设计深度

### 2. 代码可维护性
- 代码应易于理解、修改和扩展
- 关注模块化、命名清晰、结构合理

### 3. 测试覆盖
- 确保代码可验证
- 根据功能重要性自行判断测试深度
- 详见 [测试规则](.trae/rules/test_rules.md)

### 4. 文档同步
- 代码变更时，文档必须同步更新
- 详见 [文档同步规则](.trae/rules/docs_rules.md)

### 5. 安全性
- 敏感信息不外泄
- 注意依赖和配置的安全风险

## 开发流程
设计 → 实现 → 审查 → 测试 → 修复 → 再次审查 → 提交
