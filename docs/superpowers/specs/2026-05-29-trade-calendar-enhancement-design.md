# 交易日历增强设计文档

## 1. 概述
在现有交易日历基础上，为每个日期格增加日线数据条数和指标完善率显示，并将日历标识从底部圆点改为格子背景色。

## 2. 后端改动

### GET /api/trade-calendar (现有接口增强)
当传入了 `start_date` 和 `end_date` 参数时，为每条日历记录附加统计字段：

**请求**: `GET /api/trade-calendar?start_date=20260601&end_date=20260630`

**响应格式不变**，每条记录新增两个字段：
```json
[
  {
    "exchange": "SSE",
    "cal_date": "20260601",
    "is_open": 1,
    "pretrade_date": "20260531",
    "stock_count": 1500,
    "indicator_rate": 0.923
  }
]
```

### 统计计算
通过 MongoDB 聚合管道在 `stock_daily` 集合中按 `trade_date` 分组，一次查询完成：
- `stock_count`: 该日期有 OHLCV 基本数据的记录数
- `indicator_rate`: 该日期所有记录中，约 59 个指标字段的非空比例

**指标字段列表**: ma_5 ~ week_amount_avg 共 59 个非必需字段。

### 代码改动文件
| 文件 | 改动 |
|------|------|
| `data/service.py` | `get_trade_calendar_records()` 新增聚合查询逻辑 |
| `api/routers/trade_calendar.py` | 依托现有路由，无结构变化 |

## 3. 前端改动

### 日历格子布局 (方案A)
每格改为宽矩形，水平三行排列：
```
┌──────────────────────────────┐
│  日        一        二       │
├──────────────────────────────┤
│ 🟩 1        🟩 2      🟧 3   │
│    1,230条       1,235条      │
│    95.2%         94.8%        │
├──────────────────────────────┤
│ 🟩 4        🟧 5      🟩 6   │
│    1,240条  (休市)    1,238条 │
│    95.1%                 94.9% │
└──────────────────────────────┘
```

### 颜色方案 (从圆点改为背景色)
| 类型 | 背景色 | 文字颜色 |
|------|--------|----------|
| 交易日 | 浅绿 (`rgb(232, 245, 233)`) | 深色 |
| 非交易日 | 浅橙 (`rgb(255, 248, 225)`) | 灰色 |
| 无数据/非本月 | 浅灰 (`rgb(245, 245, 245)`) | 浅灰色 |

### 格子内容
- **交易日**: 日期号（大号）+ 条数（灰色小字）+ 完善率（灰色小字）
- **非交易日**: 仅日期号，显示"休市"标签
- **无数据**: 仅日期号

### 数据加载
- 前端每次切换月份/初始化时，调 `GET /api/trade-calendar?start_date=...&end_date=...`
- **不再**调用无参数的接口（新需求不再需要加载全部日历数据）
- 同步按钮行为不变

### 代码改动文件
| 文件 | 改动 |
|------|------|
| `api/tradeCalendar.ts` | 类型定义增加 `stock_count` 和 `indicator_rate` |
| `views/TradeCalendarView.vue` | 重构日历格子布局、颜色、数据绑定 |
| `router/index.ts` | 无变化 |
| `components/AppLayout.vue` | 无变化 |

## 4. 时序图
```
前端(切换月份)                后端API                 MongoDB
     │                          │                       │
     │── GET /trade-calendar ──→│                       │
     │   ?start_date=           │                       │
     │   &end_date=             │                       │
     │                          │── trade_calendar ───→│
     │                          │←── 日历记录 ─────────│
     │                          │── stock_daily聚合 ──→│
     │                          │←── 统计结果 ─────────│
     │←── 合并响应 ─────────────│                       │
     │                          │                       │
     │(渲染日历格子)             │                       │
```

## 5. 注意事项
- 无 `start_date/end_date` 时，接口保持原有行为（返回全部日历数据，不加统计）
- 聚合查询是异步的，与日历查询并行执行后合并
- 指标完善率保留 4 位小数
