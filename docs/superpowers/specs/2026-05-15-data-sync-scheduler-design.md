# 数据同步定时任务优化设计

## 背景

当前定时任务每 60 秒只处理 1 只股票，且区分 `pending` → `data_completed` → `active` 三种状态，导致：
- 数据拉取效率低
- 状态管理复杂
- 指标计算成为瓶颈

## 目标

简化流程，提高批量处理效率。

## 设计方案

### 处理流程

```
┌─────────────────────────────────────────────────────────────┐
│              每 60 秒执行一次 run_data_sync_job                │
├─────────────────────────────────────────────────────────────┤
│  1. 获取最多 300 只 pending 股票（按市值降序）               │
├─────────────────────────────────────────────────────────────┤
│  2. 逐个处理每只股票：                                       │
│     a. 拉取 20 年数据（20050101 ~ 今天）                     │
│     b. 计算指标                                              │
│     c. 更新状态为 active                                     │
│     d. 等待 0.2 秒                                          │
│     任意步骤失败 → 停止本次任务                             │
├─────────────────────────────────────────────────────────────┤
│  3. 所有股票处理完成 → 任务结束                             │
└─────────────────────────────────────────────────────────────┘
```

### 状态简化

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `active` | 同步完成，可使用 |

去掉 `data_completed` 中间状态，数据拉取完成后立即计算指标并标记为 `active`。

### 错误处理

- 遇到任何失败立即停止本次任务
- 记录错误日志
- 已处理的股票保持 `active` 状态
- 未处理的股票保持 `pending` 状态，下次任务继续

## 代码改动

### 修改文件

- `backend/src/trade_alpha/scheduler/data_sync.py`

### 函数变更

| 函数 | 变更 |
|------|------|
| `get_pending_stocks()` | `limit=1` → `limit=300` |
| `run_data_sync_job()` | 新增循环逻辑 |
| `get_data_completed_stocks()` | 删除 |
| `calculate_stock_indicators()` | 合并到主流程 |

## 预期效果

- 每分钟最多处理 300 只股票
- 流程简化，状态清晰
- 失败快速失败，不阻塞后续处理
